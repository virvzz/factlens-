# FactLens — Firefox-расширение для fact-checking через ваш AI API

FactLens — независимое Firefox WebExtension (Manifest V3), которое помогает
проверять фактические утверждения в выделенном или вставленном тексте через
**настраиваемый пользователем** AI endpoint: Anthropic, OpenAI-compatible,
DeepSeek, Artemox-like или любой собственный HTTP API.

> ⚠️ **Автоматический фактчекинг может ошибаться. Не используйте результат
> как окончательный источник истины.**
>
> Если у выбранного провайдера нет доступа к web search и актуальным
> источникам, модель обязана возвращать вердикт `UNVERIFIABLE`
> («Невозможно проверить») — это нормальное поведение, а не ошибка
> расширения.

## Возможности (Этап 4 — улучшения)

- **Sidebar** (Firefox: Alt+Shift+S; Chrome: side panel) — live-сессия активной
  вкладки, последняя проверка и история в постоянной боковой панели.
- **История проверок** — только по явному включению (настройки → «Данные»),
  локально, максимум 50 записей, очищается кнопкой.
- **Горячие клавиши**: Alt+Shift+F — popup, Alt+Shift+S — sidebar,
  Alt+Shift+X — проверить выделенный текст.
- **Справка внутри расширения** (popup → «Справка»): как подключить каждого
  провайдера и что означает каждый параметр.
- **Chrome-сборка**: `npm run build:chrome` → `dist/chrome` (загружается через
  chrome://extensions → «Режим разработчика» → «Загрузить распакованное»).
  Код общий, различия только в манифесте (service worker, side panel).
- Web-search/tool-провайдеры поддерживаются через «Дополнительные поля тела
  запроса» (пример в справке). Streaming зарезервирован: ответ — строгий JSON,
  потоковый вывод пока не даёт выгоды.

## Возможности (Этап 3 — распознавание звука)

- **Проверка видео по звуку**: расширение захватывает звук прямо из элемента
  `<video>` (Web Audio API, как расширения-эквалайзеры), пишет его кусками
  по ~20 секунд и отправляет на **STT endpoint** (Whisper-совместимый API).
  Распознанный текст идёт в тот же конвейер фактчекинга, что и субтитры.
- Настраиваемый STT-блок в настройках: пресеты **OpenAI** (`whisper-1`),
  **Groq** (`whisper-large-v3`, есть бесплатный лимит), **локальный
  Whisper-сервер**; свой auth mode, модель, язык, длина куска, response path.
- Test STT: проверка endpoint тестовым тихим фрагментом.
- Детекция тишины: куски без звука (пауза, mute) не отправляются на STT.

### Как настроить и запустить

1. Настройки FactLens → раздел **«Распознавание речи (STT)»** → кликните
   пресет (например, Groq), вставьте STT API key → **«Сохранить настройки»**
   (подтвердите разрешение на STT-домен) → **Test STT**.
2. Откройте видео, включите звук (не mute!) → иконка FactLens →
   **«🎬 Live-проверка видео»** → источник **«🎙 Распознавание звука (STT)»**.

Ограничения: DRM-видео (Netflix и т.п.) и CORS-закрытые плееры отдают
тишину — это ограничение браузера, не расширения. Звук элемента должен быть
включён (громкость системы можно убавить — важна громкость самого плеера).

## Возможности (Этап 2 — видео и субтитры)

- **Live-проверка видео**: расширение читает субтитры по мере просмотра
  (YouTube и любые страницы с HTML5-видео и дорожками субтитров), режет текст
  на фрагменты и проверяет их очередью. Результаты появляются в overlay
  поверх страницы с задержкой в несколько секунд (время ответа AI).
- **Проверка всего транскрипта**: для YouTube-видео с субтитрами можно
  проверить весь доступный транскрипт сразу.
- Overlay: статус (ожидание / чтение / проверка / ошибка), provider/model,
  карточки утверждений с цветными вердиктами, пауза/продолжить, копирование
  результатов, «сообщить об ошибке» (копирует отчёт в буфер).
- Очередь фрагментов с дедупликацией повторяющихся утверждений.

### Как запустить live-проверку

1. Откройте видео (например, YouTube) и **включите субтитры (CC)** в плеере —
   для live-режима расширение читает именно отображаемые субтитры.
2. Кликните иконку FactLens → **«🎬 Live-проверка видео (субтитры)»**.
3. В появившемся overlay выберите режим: live или весь транскрипт.
4. Смотрите видео — проверенные утверждения появляются в панели по мере
   просмотра. Кнопка ⏸ ставит проверку на паузу, ✕ — останавливает.

Ограничения: это не распознавание аудио — если у видео нет субтитров
(даже автогенерируемых), live-режим работать не будет; используйте выделение
текста или ручной ввод. Захват аудио — отдельный этап 3 (в Firefox нет
Chrome-подобного `tabCapture`, честный вариант — `getDisplayMedia` + STT).

## Возможности (Этап 1 — MVP)

- Проверка выделенного текста на странице (кнопка в popup или контекстное меню).
- Проверка вручную вставленного текста.
- Provider presets: Anthropic official, OpenAI-compatible, DeepSeek official,
  Artemox / custom OpenAI-compatible, Custom Anthropic-compatible,
  Custom raw HTTP (JSON-шаблон).
- Настраиваемые base URL, path, auth mode, model, заголовки, response parser.
- Test connection с различением типов ошибок (auth / URL / CORS / модель /
  кредиты / rate limit / parse).
- Вердикты: Верно, В основном верно, Вводит в заблуждение, Ложно,
  Невозможно проверить, Мнение / не требует фактчекинга.
- Редактируемые prompt templates с кнопкой Reset to defaults.
- Экспорт/импорт настроек **без API key**, маскирование ключа, очистка данных.
- Без аналитики, трекеров и скрытых endpoint'ов.

## Требования

- **Firefox Desktop 128+** (нужна поддержка `optional_host_permissions` в MV3).
- Node.js 20+ (только для разработки: тесты, lint, сборка).

## Почему Manifest V3 без MV2 fallback

Для сценариев MVP (запросы к API из фонового event page, popup, options)
MV3 в Firefox работает стабильно: Firefox использует **event pages**
(не service workers, как Chrome), поэтому проблем с фоновыми процессами нет.
Host permissions в Firefox MV3 запрашиваются в рантайме — это ровно та
модель, которая нужна для пользовательских endpoint'ов. MV2 fallback не
требуется. Если на этапе 3 (аудио) обнаружатся ограничения MV3, решение
будет пересмотрено отдельно.

## Установка (временная, для разработки)

1. Склонируйте/скопируйте проект.
2. Откройте в Firefox страницу `about:debugging#/runtime/this-firefox`.
3. Нажмите **«Загрузить временное дополнение…»** (Load Temporary Add-on).
4. Выберите файл `extension/manifest.json`.
5. Иконка FactLens появится на панели инструментов.

Временное дополнение удаляется при перезапуске Firefox — для постоянной
установки нужна подпись через addons.mozilla.org (вне рамок MVP).

## Установка в Chrome

1. `npm run build:chrome` (папка `dist/chrome` уже в репозитории после сборки).
2. Откройте `chrome://extensions`, включите **«Режим разработчика»**.
3. **«Загрузить распакованное расширение»** → выберите папку `dist/chrome`.
4. Sidebar в Chrome открывается через боковую панель браузера (иконка панели
   на тулбаре → FactLens); горячая клавиша Alt+Shift+S — только в Firefox.

## Быстрый старт

1. Кликните иконку FactLens → **Настройки** (или правый клик по иконке →
   Manage Extension → Preferences).
2. Выберите provider preset, укажите модель, вставьте API key.
3. Нажмите **«Сохранить настройки»** — Firefox спросит разрешение на доступ
   к домену API. **Обязательно подтвердите**: без этого разрешения запросы
   работать не будут.
4. Нажмите **Test connection** и убедитесь, что статус — «Соединение работает».
5. Выделите текст на любой странице → правый клик → «Проверить факты в
   выделенном тексте», либо откройте popup и нажмите кнопку проверки.

## Настройка провайдеров

### Anthropic official

| Поле | Значение |
|---|---|
| Preset | Anthropic official |
| Base URL | `https://api.anthropic.com` |
| Path | `/v1/messages` |
| Auth mode | `x-api-key` |
| Модель | например `claude-sonnet-4-20250514` (см. подсказки в настройках) |
| Заголовки | `{"anthropic-version": "2023-06-01"}` (подставляется автоматически) |

### OpenAI-compatible

| Поле | Значение |
|---|---|
| Preset | OpenAI-compatible |
| Base URL | `https://api.openai.com` (или URL совместимого сервиса) |
| Path | `/v1/chat/completions` |
| Auth mode | `Authorization: Bearer` |
| Модель | любая доступная у провайдера |

Подходит и для локальных OpenAI-compatible серверов (Ollama, LM Studio и
т.п.): укажите `http://localhost:<порт>` и auth mode «Без авторизации».

### DeepSeek official

| Поле | Значение |
|---|---|
| Preset | DeepSeek official |
| Base URL | `https://api.deepseek.com` |
| Path | `/chat/completions` |
| Auth mode | `Authorization: Bearer` |
| Модель | `deepseek-v4-flash` или `deepseek-v4-pro` |

`deepseek-chat` и `deepseek-reasoner` доступны как legacy-варианты — при их
выборе настройки покажут предупреждение. Reasoning/thinking-параметры можно
задать в секции «Дополнительные поля тела запроса» (JSON добавляется в тело
запроса как есть).

### Artemox / custom OpenAI-compatible

| Поле | Значение |
|---|---|
| Preset | Artemox / custom OpenAI-compatible |
| Base URL | `https://api.artemox.com/v1` |
| Path | `/chat/completions` |
| Auth mode | `Authorization: Bearer` |

Base URL уже содержит `/v1` — расширение корректно склеивает URL и не
удваивает сегменты (`https://api.artemox.com/v1/chat/completions`).

### Custom Anthropic-compatible / Custom raw HTTP

- **Custom Anthropic-compatible**: свой base URL + формат Anthropic Messages.
- **Custom raw HTTP**: полностью свой JSON-шаблон тела запроса с переменными
  `{{model}}`, `{{system_prompt}}`, `{{user_prompt}}`, `{{max_tokens}}`,
  `{{temperature}}`, `{{language}}`, `{{strictness}}` и свой response parser
  path (например `data.output[0].text`). Строковые переменные указывайте в
  кавычках — значения экранируются автоматически.

## Проверка соединения

Кнопка **Test connection** отправляет короткий тестовый запрос и различает:
успех, неверный ключ (401/403), несуществующий endpoint (404), сетевую
ошибку/CORS/отсутствие разрешения, несуществующую модель, нехватку
средств/квоты (402, `insufficient_quota`), rate limit (429) и ошибки парсинга
ответа.

## Как удалить ключ и данные

- **Удалить ключ**: Настройки → раздел «Авторизация» → «Удалить ключ».
- **Очистить всё**: Настройки → раздел «Данные» → «Очистить все локальные
  данные» (настройки, ключ, последний результат).

## Какие данные и куда отправляются

- Проверяемый текст (выделенный или вставленный) и prompt-инструкции
  отправляются **только** на endpoint, который вы сами указали в настройках.
  Точный адрес всегда показан в настройках («Текст будет отправляться на: …»)
  и в подсказке popup.
- API key отправляется только на тот же endpoint в auth-заголовке.
- Больше никуда ничего не отправляется: нет аналитики, телеметрии,
  «домашнего» сервера. История проверок не сохраняется (последний результат
  хранится в памяти сессии браузера и исчезает при закрытии Firefox).
- Настройки, включая API key, хранятся локально в `browser.storage.local`.
  Это **не** полноценный менеджер секретов: данные лежат в профиле Firefox
  без шифрования. Экспорт настроек никогда не включает ключ.

## Ограничения fact-checking без web search

Обычный chat-API (без tools/web search) проверяет утверждения только по
знаниям модели, у которых есть дата среза. Поэтому:

- события после среза знаний модели получат `UNVERIFIABLE`;
- модель проинструктирована **не выдумывать источники** — пустой список
  источников лучше вымышленного;
- вердикты — это предварительная оценка, а не вердикт судьи. Всегда
  проверяйте важные утверждения по первоисточникам.

## Разработка

```bash
npm install     # зависимости (web-ext)
npm run lint    # web-ext lint (0 ошибок, 0 предупреждений)
npm test        # unit-тесты (node --test)
npm run build   # сборка zip в dist/
npm run zip     # то же самое
```

### Структура проекта

```text
extension/
  manifest.json            # MV3, минимальные permissions
  icons/icon.svg
  src/
    background/
      background.js        # event page: меню, роутер сообщений, permissions
      apiClient.js         # fetch + таймауты/ретраи + классификация ошибок
      providerAdapters/
        anthropicMessages.js
        openaiChatCompletions.js
        deepseekOfficial.js
        customTemplate.js
      prepare.js           # валидация настроек + host permission перед запросом
      liveSession.js       # очередь live-фрагментов по вкладкам
    audio/
      capture.js           # захват звука из <video> (Web Audio + MediaRecorder)
      speechToText.js      # STT-клиент (Whisper-совместимый API)
    content/
      contentLib.js        # чистые функции: склейка субтитров, чанкер, timedtext
      contentScript.js     # оркестратор live/transcript режимов
      overlay.js           # overlay (Shadow DOM)
      subtitleExtractors/
        youtube.js         # live-субтитры из DOM + транскрипт (timedtext)
        genericCaptions.js # HTML5 <video> textTracks
    popup/                 # popup UI
    options/               # страница настроек
    shared/
      settings.js          # дефолты, presets, миграция, экспорт/импорт
      prompts.js           # редактируемые prompt templates
      resultParser.js      # извлечение JSON, нормализация claims
      validators.js        # joinUrl, маска ключа, getByPath
      sanitizer.js         # очистка текста, redaction секретов
      logger.js            # логгер, никогда не пишущий ключи
      auth.js, errors.js
      i18n/ru.js, i18n/en.js
tests/                     # unit-тесты (node --test)
```

### Permissions

- `storage`, `activeTab`, `scripting`, `contextMenus` — базовые.
- `host_permissions` пустые; доступ к API-домену запрашивается в рантайме
  через `optional_host_permissions` только для того origin, который вы
  сохранили в настройках.

### Тесты

Покрыто: склейка base URL + path (включая дедупликацию `/v1`), маскирование
ключа, сборка запросов Anthropic/OpenAI/DeepSeek/custom, парсеры ответов,
обработка HTTP 401/403/402/404/429/500, извлечение JSON из «болтливых»
ответов модели, фильтрация claims, миграция настроек, исключение ключа при
экспорте, redaction секретов в логах.
