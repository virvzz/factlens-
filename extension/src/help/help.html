FactLens — справка

Автоматический фактчекинг может ошибаться. Не используйте результат как окончательный источник истины. Вердикты — предварительная оценка AI-модели, которая помогает быстрее заметить спорные утверждения.

Быстрый старт

  1. Откройте Настройки (иконка FactLens → «Настройки»).
  2. Выберите Provider preset — поля endpoint заполнятся сами:
    • Anthropic official — прямой ключ Anthropic (ключи sk-ant-…);
    • OpenAI-compatible — OpenAI или любой совместимый сервис, включая локальные (Ollama, LM Studio);
    • DeepSeek official — api.deepseek.com;
    • Artemox / custom OpenAI-compatible — прокси-сервисы вида api.artemox.com;
    • Custom Anthropic-compatible / Custom raw HTTP — любой свой endpoint.
  3. Впишите модель (кликните подсказку под полем) и вставьте API key.
  4. Нажмите «Сохранить настройки» и обязательно подтвердите разрешение — браузер спросит доступ к домену API. Без этого запросы не пойдут.
  5. Нажмите Test connection — должно быть «Соединение работает».
  6. Для проверки звука видео дополнительно настройте раздел STT.

Результаты live-режимов показываются в панели поверх страницы (новые сверху), там же — пауза ⏸, копирование ⧉ и «Сообщить об ошибке» на каждой карточке. Боковая панель (Alt+Shift+S) показывает текущую сессию, последнюю проверку и историю.

Параметры провайдера

ПараметрЧто означает
Provider presetГотовый набор настроек под провайдера. Смена пресета перезаписывает поля endpoint (ключ не трогает).
Base URLАдрес сервера API, например https://api.artemox.com/v1. Завершающие «/» и повтор /v1 обрабатываются автоматически.
API pathПуть метода на сервере: /chat/completions (OpenAI-формат) или /v1/messages (Anthropic-формат). Итоговый адрес показан зелёной строкой под полями.
Request formatФормат тела запроса: Anthropic Messages, OpenAI Chat Completions или свой JSON-шаблон (Custom raw HTTP).
Response parser pathГде в JSON-ответе лежит текст модели: choices[0].message.content (OpenAI) или content[0].text (Anthropic). Менять нужно только для нестандартных API.
МодельИмя модели у провайдера. Если провайдер отклоняет модель («Guardrail», «model not found») — запросите список: GET {base URL}/models с вашим ключом.
Auth modeКак передаётся ключ: заголовок Authorization: Bearer (OpenAI-совместимые), x-api-key (Anthropic), свой заголовок или без авторизации (локальные серверы).
API keyСекретный ключ провайдера. Хранится локально, показывается только маской, не попадает в экспорт и логи. Пустое поле при сохранении = оставить прежний ключ.
Дополнительные заголовкиJSON-объект заголовков, например {"anthropic-version": "2023-06-01"}.
Дополнительные поля тела запросаJSON-объект, который добавляется в тело каждого запроса. Сюда идут provider-specific параметры: reasoning-опции DeepSeek, инструменты и т.п. Например, web search для Anthropic official: {"tools": [{"type": "web_search_20250305", "name": "web_search", "max_uses": 3}]} (работает только если провайдер поддерживает tools; прокси обычно не пропускают).

Параметры генерации

ПараметрЧто означает
Max tokensМаксимальная длина ответа модели. 1024 хватает на ~10 утверждений; если ответы обрываются (ошибка parse) — увеличьте.
Temperature«Творческость» модели, 0–2. Для фактчекинга держите низкой (0.0–0.3): стабильнее JSON и меньше выдумок.
Таймаут, секСколько ждать ответа до обрыва запроса.
Повторы при сбоеСколько раз повторять запрос при временных ошибках (сеть, 429, 5xx). Ошибки ключа/модели не повторяются.
Пауза между повторамиЗадержка перед повтором, сек.
Язык ответаНа каком языке модель пишет объяснения: авто (язык текста), русский, английский или свой.
Строгость фактчекингаМягкая — помечает только явную ложь; сбалансированная — обычные редакционные стандарты; строгая — сомнительное скорее получит «Невозможно проверить», чем догадку.
StreamingЗарезервировано: ответ — строгий JSON, который можно разобрать только целиком, поэтому потоковый вывод пока не даёт выгоды. Настройка сохраняется, но не используется.

Распознавание речи (STT)

Отдельный сервис, который превращает звук в текст (модели Claude/GPT/DeepSeek этого не делают). Нужен только для режима «🎙 Распознавание звука».

ПараметрЧто означает
ПресетыOpenAI (whisper-1, платно ~$0.006/мин), Groq (whisper-large-v3, есть бесплатный лимит — ключ на console.groq.com), Локальный Whisper (бесплатно, свой сервер, авторизация не нужна).
STT base URL / pathEndpoint Whisper-совместимого API, обычно /v1/audio/transcriptions.
STT auth mode / API keyКак и у основного провайдера; ключ тоже маскируется и не экспортируется.
STT модельwhisper-1 (OpenAI), whisper-large-v3 (Groq) и т.п.
Язык аудиоКод языка (ru, en…) — подсказка распознавателю. Пусто = автоопределение.
Длина куска, секКак часто звук отправляется на распознавание (по умолчанию 20 с). Меньше — быстрее реакция, но больше запросов; больше — дешевле и точнее контекст.
STT response pathГде в ответе лежит текст; для Whisper-совместимых API — text.
Test STTОтправляет короткий тихий фрагмент и проверяет, что endpoint отвечает.

Как работает: звук берётся из самого элемента видео (Web Audio API), пишется кусками, тишина отфильтровывается локально и на STT не отправляется. Звук плеера должен быть включён — убавляйте громкость системой, а не кнопкой mute.

Промпты

Инструкции, которые получает модель. Редактируются в настройках («Prompt templates»), кнопка Reset to defaults возвращает исходные. Основные: Fact-checking (system) — правила проверки и схема JSON; Fact-checking (user) — обёртка проверяемого текста (переменная {{text}}). Остальные зарезервированы под будущие режимы. В system-промпте можно менять правила, но не убирайте требования «строго JSON» и «не выдумывать источники» — на них держится разбор ответа.

ВердиктСмысл

В основном верноВерно по сути, неточно в деталях.
Вводит в заблуждениеФормально не ложь, но создаёт ложное впечатление.
ЛожноПротиворечит известным фактам.

Стоимость и как её снизить

Расширение платит вашими ключами, поэтому стоимость = тарифы провайдера. Ориентир: 13-минутное видео на claude-sonnet ≈ $0.7. Что помогает:

  • Дешёвая модель для live: claude-haiku-4-5 в разы дешевле sonnet — для потоковой проверки её обычно достаточно; sonnet оставьте для разовых важных проверок.
  • Длина куска STT 30–45 сек: меньше запросов к AI (каждый кусок = отдельный запрос с полным системным промптом).
  • Пауза ⏸ на неинформативных участках (музыка, реклама).
  • Субтитры вместо STT, если они есть: сам STT тоже стоит денег (у Groq — бесплатный лимит).
  • Сокращение system-промпта тоже экономит, но осторожно — не сломайте правила JSON.

Типичные ошибки

СообщениеЧто делать
Ошибка авторизацииПроверьте ключ (маска сохранённого — под полем) и auth mode. В поле ключа не должно быть пробелов и лишнего текста.
Модель не найдена / отклоненаИмя модели с опечаткой или недоступно на тарифе. Запросите GET {base URL}/models.
Нет разрешения на …Нажмите «Сохранить настройки» и подтвердите запрос браузера на доступ к домену.
Endpoint не найден (404)Проверьте base URL и path; итоговый адрес виден под полями.
Недостаточно средств/квотыПополните баланс у провайдера.
Превышен лимит (429)Подождите; для live увеличьте длину куска.
Не удалось извлечь JSONМодель ответила не по формату; расширение само повторяет запрос с напоминанием. Если ошибка постоянная — уменьшите temperature, увеличьте max tokens или смените модель.
Звук не захватываетсяВидео на паузе, mute, либо DRM/CORS-защита плеера. Используйте субтитры или выделение текста.

Приватность

  • Проверяемый текст и аудио уходят только на endpoint'ы, которые вы сами указали (адреса всегда видны в настройках).
  • Нет аналитики, трекеров, серверов разработчика.
  • История проверок по умолчанию выключена; включается в настройках → «Данные», хранится локально (макс. 50 записей), очищается кнопкой.
  • Ключи хранятся в storage.local (профиль браузера, без шифрования — это не менеджер секретов), маскируются в интерфейсе и никогда не попадают в экспорт настроек и логи.

Ограничения

  • Без web search модель проверяет только по своим знаниям (с датой среза) — события позже среза получат «Невозможно проверить».
  • Модель проинструктирована не выдумывать источники; пустой список источников — честный ответ.
  • DRM-видео и CORS-закрытые плееры не дают доступ к звуку — ограничение браузера.
  • Задержка live-режимов = накопление фрагмента + распознавание (для STT) + ответ AI, обычно 20–40 секунд.
FactLens — независимая реализация. Не является окончательным источником истины; проверяйте важные утверждения по первоисточникам.
